# SCRAPE-REPORT.md

Phase 0 output — assets pulled from the legacy WordPress + Elementor site.

## Sources crawled

- `https://zagagsheli.co.il/` (home)
- `https://zagagsheli.co.il/examples/` (gallery)
- `https://zagagsheli.co.il/wp-json/wp/v2/media` (WP REST, paged)

The WP REST route bypassed Elementor's "load more" lazy-load and returned the full
media library in a single sweep.

## Categorization

Two passes:

1. **Filename heuristics + seed list** from the brief (18 known images across
   shower / mirrors / brand). Weak signal for the rest (filenames are just
   phone-camera timestamps like `IMG_20191216_121744-1.jpg`).
2. **Elementor gallery filter classes** — the examples page renders each item
   inside `<div class="eael-cf-<HEBREW_TAG>">`. Decoded the HTML-encoded
   `data-gallery-items` blob and mapped every image to its declared Hebrew
   category (`מקלחונים-עם-פרזול-שחור` → shower, `מראות` → mirrors, etc.).

The second pass reclassified 38 images from `uncategorized` to their true bucket.

## Orphans (37 unclassified)

These are in `public/images/raw/uncategorized/`. Likely candidates:

- Old blog-post images that predate the current gallery (WhatsApp forwards, screenshots)
- Images uploaded but never inserted into any Elementor widget
- Featured images for pages that no longer exist

A local review sheet is generated by `npm run contact-sheet` at
`public/images/raw/_review.html`. Owner can eyeball the orphans and either
delete them or hand-move into the correct category subfolder — then re-run
`npm run optimize && npm run db:seed` to pull them into the DB.

**Not blocking**: the categorized 63 images plus brand assets are enough to
build a rich gallery. Missing categories (bath, custom) have zero images and
will show placeholder-friendly empty states with a CTA prompting Lior to
upload real work from his phone via the admin panel.
